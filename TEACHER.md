# 가이드(선생님) 기능 명세

> 선생님(가이드)이 사용하는 각 기능의 설계 의도와 동작 방식을 정의하는 문서.
>
> 📌 **함께 읽기**: [UX_FLOW.md](./UX_FLOW.md) — 화면 경로·IA 배치 기준 · [MISSION_DESIGN.md](./MISSION_DESIGN.md) — 미션 설계 원칙
> 각 기능의 **화면 경로**(`/teacher/notices` 등)와 탭 배치 기준은 UX_FLOW.md에서 정의합니다.

---

## 설계 철학

> **"선생님이 수업에 집중할 수 있도록, 관리 부담을 최소화한다."**

- 복잡한 설정 없이 바로 쓸 수 있어야 함
- 소통은 일방향 공지 중심 (개인 연락 최소화)
- 빌더(학생)의 동기부여 상태를 한눈에 파악 가능해야 함

---

## 기능별 명세

### 📋 퀘스트 관리 (`/teacher/missions`)

**목적**: 선생님이 빌더에게 줄 퀘스트를 만들고 관리.

| 항목 | 내용 |
|------|------|
| 퀘스트 생성 | 제목·내용·담당 캐릭터·완료 방식·코인 보상 설정 |
| 완료 방식 | 즉시완료 / 퀴즈 / 가이드 확인 3가지 |
| 퀘스트 상태 | 활성 / 완료 / 예정 |
| 담당 캐릭터 | 선택 시 완료 때 해당 캐릭터 친밀도 +1 |

**설계 의도**: 미션이 "학습 과제"처럼 보이지 않게, 캐릭터와 연결해서 게임 속 이야기처럼 느껴지게 함. → [MISSION_DESIGN.md](./MISSION_DESIGN.md) 참고.

---

### ✅ 승인 관리 (`/teacher/approvals`)

**목적**: "가이드 확인" 방식 퀘스트의 완료 요청을 처리.

| 항목 | 내용 |
|------|------|
| 승인 대기 목록 | 빌더가 제출한 완료 요청 목록 |
| 처리 | 승인(코인 지급) / 반려 |
| 우선순위 | 대시보드에서 즉각 확인 필요 항목으로 노출 |

**설계 의도**: 선생님이 수업 중 흘끔 보고 처리할 수 있을 만큼 가볍게.

---

### 👥 빌더 관리 (`/teacher/classes`)

**목적**: 반 빌더(학생) 현황 파악 및 코인 수동 조정.

| 항목 | 내용 |
|------|------|
| 목록 | 빌더별 코인 잔액, 보유 주민 수 |
| 상세 | 완료 퀘스트 이력, 보유 주민 목록 |
| 코인 수동 지급/차감 | 특별 이벤트, 격차 보정용 |

**설계 의도**: 선생님이 격차를 파악하고 수동으로 보정할 수 있어야 함. 자동화만으론 교실 상황을 반영 못함.

---

### 📢 알림장 (`/teacher/notices`)

**목적**: 선생님 → 빌더(학생) 단방향 공지.

| 항목 | 내용 |
|------|------|
| 작성 | 제목 + 내용 입력 |
| 발송 | 반(월드) 전체 대상 |
| 열람 | 빌더 홈 화면에 알림장 섹션으로 노출 |

**설계 의도**: 선생님들은 개인 연락보다 공지를 선호함. 전화·메시지 없이 앱 안에서 전달 완료. → 일방향 소통이 핵심.

---

### 🗺️ 자리배치도 (`/teacher/tools`)

**목적**: 교실 자리 배치를 시각적으로 관리.

| 항목 | 내용 |
|------|------|
| 배치 방식 | 빌더 이름 드래그앤드롭 |
| 저장 | Supabase 실시간 저장 |
| 연동 | 빌더 계정과 연결됨 |

**설계 의도**: 학기 초 자리 배치 + 이후 조정을 빠르게 처리.

---

### 🔗 QR코드 초대 (`/teacher/qrcodes`)

**목적**: 빌더(학생)를 월드(반)에 초대.

| 항목 | 내용 |
|------|------|
| 생성 | 반별 고유 QR 자동 생성 |
| 사용 | 학생이 QR 스캔 → 앱 진입 → 계정 생성 |

**설계 의도**: 설치 없이(PWA), QR 하나로 온보딩 완료.

---

### 📖 가이드 온보딩 (`/teacher/guide`)

**목적**: 처음 쓰는 선생님을 위한 기능 설명 화면.

| 항목 | 내용 |
|------|------|
| 내용 | 주요 기능 순서별 안내 |
| 진입 | 사이드바 하단 고정 |

## 현장 인사이트

실제 교사 인터뷰 및 준엽의 현장 관찰 기반.

| 인사이트 | 기능 반영 |
|---------|---------|
| 선생님은 개인 연락보다 일방향 공지를 선호 | 알림장 기능 핵심으로 설계 |
| 수업 중 앱 확인 시간은 매우 짧음 | 승인·관리 UI를 최대한 간결하게 |
| 학생 간 격차 보정이 필요한 순간이 생김 | 코인 수동 지급/차감 기능 유지 |
| 배포 전략 (2026-02-25 준엽) | 학급관리+안내 시스템 → 개인농장 → 학급 마을 순서로 완성 후 배포 |
