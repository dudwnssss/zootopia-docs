# IA_PRINCIPLES.md — 정보 구조 설계 기준

> Zootopia 앱의 기능이 어느 탭/섹션에 위치해야 하는지, 순서는 어떻게 정해야 하는지에 대한 공식 기준.
> 새 기능 추가 시 반드시 이 문서를 먼저 참고할 것.

---

## 1. 기본 철학

### 사용자별 멘탈 모델
- **빌더(학생)**: "나는 지금 무엇을 해야 하나?" → 미션 중심, 보상 중심
- **가이드(선생님)**: "나는 지금 무엇을 관리해야 하나?" → 관리 중심, 소통 중심

기능 배치는 이 멘탈 모델과 일치해야 한다.
사용자가 그 기능을 쓸 때 머릿속에서 어떤 맥락에 있는지를 먼저 생각할 것.

---

## 2. 탭바 vs 페이지 내 섹션 결정 기준

### 탭바에 올라오는 기능의 조건 (모두 충족해야 함)
1. **독립성** — 다른 탭과 콘텐츠가 겹치지 않는다
2. **전환 빈도** — 하루에 여러 번 접근할 가능성이 있다
3. **진입점 역할** — 그 기능이 여러 하위 화면의 시작점이 된다

### 페이지 내 섹션으로 들어가는 기능의 조건
1. 해당 기능이 상위 탭의 **맥락 안에서만** 의미가 있다
2. 독립적으로 쓰이기보단, 특정 화면을 열었을 때 **부가적으로** 확인하는 정보다
3. 사용 빈도가 낮아서 상단 탭을 차지할 가치가 없다

### 예시
| 기능 | 위치 | 이유 |
|------|------|------|
| 알림장 | 가이드 사이드바 탭 | 매일 작성 가능. 소통 기능으로 독립성 있음. 미션/빌더 관리와 맥락 다름. |
| 퀘스트 승인 | 별도 탭 (승인) | 미션 탭과 관련 있지만, 처리 주기가 달라서 독립 탭이 효율적 |
| 자리배치 | 도구 탭 내부 | 자주 쓰지 않음. 도구의 일부. 독립 탭이 필요할 정도로 진입 빈도 높지 않음. |
| 빌더 상세 | 빌더 관리 탭 내부 | 빌더 목록 → 클릭 → 상세. 독립 탭이 아닌 드릴다운 구조가 적절. |

## 3. 가이드(선생님) 사이드바 순서 원칙

### 현재 순서
```
홈 🏠 → 월드 관리 🏫 → 미션 🎯 → 승인 📋 → 알림장 📢 → QR코드 🔗 → 도구 🔧
```

### 순서 결정 기준

#### 원칙 1: 업무 플로우 순서
선생님이 수업을 시작할 때 하는 행동 순서를 반영한다.
```
전체 현황 파악 → 반/학생 확인 → 미션 관리 → 완료 승인 → 소통(알림장) → 초대(QR) → 기타 도구
```

#### 원칙 2: 사용 빈도 순서 (동일 단계면 더 자주 쓰는 것이 위)
- 매일 쓰는 것 → 위쪽
- 주 1회 이하 → 아래쪽

#### 원칙 3: 관리 기능 → 소통 기능 → 설정/도구 순
| 카테고리 | 포함 기능 |
|---------|---------|
| 관리 | 홈(대시보드), 월드(반/학생), 미션, 승인 |
| 소통 | 알림장 |
| 접근 | QR코드 |
| 도구 | 자리배치, 기타 |

### 새 기능 추가 시 사이드바 위치 결정 흐름
```
1. 카테고리 분류: 관리 / 소통 / 접근 / 도구 중 하나?
2. 탭 자격 심사: 독립성 + 전환빈도 + 진입점 역할 모두 충족?
   - NO → 기존 탭 내 섹션으로 편입
   - YES → 3번으로
3. 순서 결정: 같은 카테고리의 기존 탭 중 업무 플로우상 언제 쓰이는가?
```

## 4. 빌더(학생) 하단 탭바 순서 원칙

### 현재 순서
```
홈 🏠 → 퀘스트 🎯 → 스카우트 🧭 → 도감 📖 → 랭킹 🏆
```

### 순서 결정 기준

#### 핵심 루프 반영
```
홈(현황) → 퀘스트(할 일) → 스카우트(보상 사용) → 도감(결과 확인) → 랭킹(동기부여)
```

빌더의 하루 루틴이 왼쪽에서 오른쪽으로 자연스럽게 흐르도록 배치.

#### 탭 자격 기준 (가이드와 동일)
1. 독립성 있는가?
2. 하루에 여러 번 전환할 만한가?
3. 여러 하위 화면의 진입점인가?

### 예시
| 기능 | 위치 | 이유 |
|------|------|------|
| 월드미션(레이드) | 퀘스트 탭 내 섹션 또는 홈 섹션 | 빈도가 불규칙. 별도 탭보다 기존 탭 안에서 접근이 자연스러움. |
| 트레이드 | 탭 여부 미정 — 기획 확정 필요 | 사용 빈도, 독립성 아직 검증 안 됨 |
| 프로필 | 탭 외부(홈 상단 아이콘 등) | 자주 전환하는 기능 아님. 탭바 점유 비효율. |

## 5. 화면 내 섹션 순서 원칙

### 홈 화면 (빌더)
```
상태 요약 (코인, 마을 레벨) → 긴급/활성 퀘스트 → 알림장 → 기타
```
- 위쪽: 지금 당장 해야 할 것
- 아래쪽: 참고용 정보

### 대시보드 (가이드)
```
빌더 현황 요약 → 대기 중인 승인 → 최근 알림장 → 기타 통계
```
- 위쪽: 즉각 처리 필요한 것 (승인 대기 등)
- 아래쪽: 정보성 요약

### 원칙 요약
| 위치 | 콘텐츠 성격 |
|------|-----------|
| 최상단 | 현재 상태/요약 (수치, 현황) |
| 중단 | 즉각 행동이 필요한 것 (할 일, 대기 중) |
| 하단 | 참고 정보, 소통, 히스토리 |

## 6. 새 기능 추가 체크리스트

기능을 새로 추가할 때 반드시 답할 것:

- [ ] **대상**: 빌더용? 가이드용? 둘 다?
- [ ] **카테고리**: 관리 / 소통 / 접근 / 도구 / 게임 중 하나
- [ ] **독립성**: 다른 기능과 콘텐츠 겹침 없이 단독으로 존재 가능한가?
- [ ] **빈도**: 하루에 몇 번? (여러 번 → 탭 후보 / 가끔 → 서브 섹션)
- [ ] **진입점**: 이 기능이 다른 화면들의 시작점인가?
- [ ] **업무 플로우상 위치**: 사용자의 하루 루틴에서 언제 이 기능을 쓰는가?
- [ ] **탭 OR 섹션**: 위 기준으로 탭 자격 있는가?
- [ ] **탭이라면 순서**: 기존 탭들 사이에서 어디에 끼어야 흐름이 자연스러운가?
